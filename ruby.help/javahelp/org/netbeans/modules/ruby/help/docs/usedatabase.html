Using Database Servers With JRuby

You choose the database server when you create a Ruby on Rails project in the Database Configuration dialog box. To create a Ruby on Rails project, choose File> New Project, select Ruby in the Category list and Ruby on Rails Application in the Projects list.

Choosing the Database Server for a Project

Using JRuby means that you have a number of choices in the NetBeans IDE for the database server. You can access a database with JRuby two ways, with or without JDBC. If you use JDBC, NetBeans supplies the drivers for MySQL and PostgreSQL. If you do not use JDBC, you can use adapters as you would in a native Ruby application.

For the latest information about database support in NetBeans Ruby, see "Using Database Servers With JRuby" in .

Registering a Database

You do not need to register your database server with the IDE to use it from your Ruby project. After a database is registered, you can create and view tables and exercise SQL commands directly from the IDE.

You can register a database in NetBeans at any time by using the Database Configuration dialog box. To do so, click on the Services tab, right click on Databases, and click New Connection. You can also quickly register a database in the Database Configuration dialog box when creating your Rails project by selecting that option in the Development, Test, or Production drop-down boxes. If you are using JavaDB, you do not need to register it. 

Adding a Database

You add a database in the Database Configuration dialog when creating your Rails project.

  1. Choose File > New Project, select Ruby in the Category list and Ruby on Rails Application in the Projects list, and click Next twice.
  2. Select Configure Using IDE Connections
  3. Click the Create DB button. A dialog box will open enabling you to create a MySQL database.
  4. After creating a database, type the connection parameters.

The newly created connection is then preselected in the associated database drop-down box. See the Registering a Database section above for information about how to register a database prior to or after creating your Rails project.
