New Ruby on Rails Application Wizard: Name and Location

See Also 

In this panel of the New Ruby on Rails Application wizard, you set the name of the project and can change its default location. You can also make choices concerning the database to use and whether or not to use Rake.

You get to this panel by choosing File > New Project, and, in the New Project wizard, selecting the category Ruby and the project type Ruby on Rails Application.

  • Clicking the Next button will take you to the next panel, which enables you to install or update Rails and to install OpenSSL support. Clicking the Finish button creates the project without these options.

The panel displays the following options:

  • Project Name. Specifies the name of the project, which is displayed in the Projects window. This name is also used to create the project folder and the context path.
  • Project Location. Specifies the location where you want to store the project metadata and source code.
  • Project Folder. Specifies the folder where you want to store the project metadata and source code. The folder is a concatenation of the project location and the project name.
  • Set as Main Project. Sets the newly created project as the main project in the Projects window.
  • Database. Specifies the database you will use with the project. If you want use a database other than one of those on the list, choose Other. When the project starts up, your database.yml file automatically opens so you can edit it.
  • If Using JRuby. The following two options are useful only if you are using JRuby.
    • Access Database Using JDBC. Specifies that your application will use the ActiveRecord-JDBC gem, which enables your application to access database servers that provide JDBC 3.0 compliant drivers.
      • You typically do not select this option if you are using MySQL, since NetBeans Ruby supplies an adapter for that database that works without JDBC.
      • If you choose this option, you must copy the driver file for the database that you choose to the JRuby lib directory. The path to this directory is similar to netbeans-install-dir/ruby1/jruby-1.0/lib. To find the location of your JRuby installation, choose Tools > Options from the main menu, click Miscellaneous, and click the Ruby icon.

      For more information, see "Using Database Servers With JRuby" in

    • Add Rake Targets to Support App Server Deployment (.war). Choose this option if you want to create a .war file so you can deploy your application to an application server, such as Glassfish.
